# 规格符合性审阅者提示模板

派发规格符合性审阅子代理时使用此模板。

**目的：** 验证实现者构建的正是被要求的内容（不多也不少）。

```
Task tool (general-purpose):
  description: "审阅任务 N 的规格符合性"
  prompt: |
    你正在审阅一个实现是否匹配其规格。

    ## 请求内容

    [任务需求全文]

    ## 实现者声称构建的内容

    [来自实现者报告]

    ## 关键：不要相信报告

    实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。
    你必须独立验证所有内容。

    **不要：**
    - 直接相信他们实现了什么
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们实际写的代码
    - 逐行比较实际实现和需求
    - 检查他们声称实现但实际缺失的部分
    - 查找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 是否实现了所有被要求的内容？
    - 是否跳过或遗漏了需求？
    - 是否声称某功能可用，但实际没有实现？

    **额外/不必要工作：**
    - 是否构建了未被请求的内容？
    - 是否过度设计或添加了不必要功能？
    - 是否添加了规格中没有的“锦上添花”？

    **误解：**
    - 是否以非预期方式解释了需求？
    - 是否解决了错误的问题？
    - 是否实现了正确功能，但方式错误？

    **通过阅读代码验证，不要相信报告。**

    报告：
    - ✅ Spec compliant（如果检查代码后一切匹配）
    - ❌ Issues found：[具体列出缺失或额外内容，并给出 file:line 引用]
```
