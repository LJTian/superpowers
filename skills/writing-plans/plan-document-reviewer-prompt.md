# 计划文档审阅者提示模板

派发计划文档审阅子代理时使用此模板。

**目的：** 验证计划是否完整、是否匹配规格，并且任务拆分是否合理。

**派发时机：** 完整计划写完之后。

```
Task tool (general-purpose):
  description: "审阅计划文档"
  prompt: |
    你是计划文档审阅者。验证这个计划是否完整，并已准备好实现。

    **要审阅的计划：** [PLAN_FILE_PATH]
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、未完成任务、缺失步骤 |
    | 规格一致性 | 计划覆盖规格需求，没有严重范围蔓延 |
    | 任务拆分 | 任务边界清晰，步骤可执行 |
    | 可构建性 | 工程师能否按此计划执行而不被卡住 |

    ## 校准

    **只标出会在实现阶段造成真实问题的事项。**
    实现者做错东西或被卡住，才是问题。
    轻微措辞、风格偏好和“锦上添花”建议不是问题。

    除非存在严重缺口，否则批准：规格需求缺失、步骤矛盾、占位内容，
    或任务过于模糊以至于无法执行。

    ## 输出格式

    ## 计划审阅

    **状态：** Approved | Issues Found

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么它会影响实现]

    **建议（仅供参考，不阻塞批准）：**
    - [改进建议]
```

**审阅者返回：** 状态、问题（如有）、建议
